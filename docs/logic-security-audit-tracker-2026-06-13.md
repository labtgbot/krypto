# Трекер остаточного аудита логики и безопасности (2026-06-13)

Дата аудита: 2026-06-13

Запрос-источник: https://github.com/labtgbot/krypto/issues/137
Pull request аудита: https://github.com/labtgbot/krypto/pull/138
Полный отчет: [`docs/logic-security-audit-2026-06-13.md`](logic-security-audit-2026-06-13.md)

## Назначение

Issue #137 просит снова полностью проанализировать логику приложения, проверить
все тщательно и оформить найденные flaws/bugs/vulnerabilities как отдельные
профессиональные задачи с тегами и стадиями реализации. Этот документ хранит
структуру третьей остаточной волны в репозитории, чтобы номера задач, коды SEC,
severity, milestone и связь с отчетом были версионируемыми и проверяемыми тестом.

Предыдущие волны: SEC-01..SEC-19 (#88-#106) и SEC-20..SEC-23 (#129-#132, PR #128).
Новая волна продолжает нумерацию с SEC-24 и фокусируется только на подтвержденных
в коде остаточных дефектах, не пересекающихся с уже заведенными задачами.

## Соглашение о метках

Каждая подзадача несет три метки:

- `security` — направление аудита.
- `severity: <high|medium|low>` — оценка серьезности находки.
- `audit-2026-06` — принадлежность к июньской audit-wave.

Метки и milestone заданы при заведении задач и должны сохраняться до закрытия.
