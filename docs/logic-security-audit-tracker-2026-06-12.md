Pull request аудита: https://github.com/labtgbot/krypto/pull/128
Полный отчет: [`docs/logic-security-audit-2026-06-12.md`](logic-security-audit-2026-06-12.md)

## Назначение

Issue #127 просит повторно проверить всю логику приложения, изучить закрытые
issues/PR и оформить найденные flaws/bugs/vulnerabilities как отдельные
профессиональные задачи с тегами и стадиями. Этот документ хранит структуру
новой остаточной волны в репозитории, чтобы номера задач, коды SEC, severity,
milestone и связь с отчетом были версионируемыми и проверяемыми тестом.

Предыдущая audit-wave SEC-01..SEC-19 заведена в #87-#106 и закрыта отдельными PR.
Новая волна продолжает нумерацию с SEC-20 и фокусируется только на
подтвержденных остаточных дефектах текущего кода.

## Соглашение о метках

Каждая подзадача несет три метки:

- `security` — направление аудита.
- `severity: <high|medium>` — оценка серьезности находки.
- `audit-2026-06` — принадлежность к июньской audit-wave.

Метки и milestone заданы при заведении задач и должны сохраняться до закрытия.
